## Data Sources & Invariants
- Source of truth for target project path:
  - `--path` flag when set.
  - otherwise current working directory.
- Source of truth for symbol input: first positional argument.
- Invariants:
  - Command signature remains `cs refs <symbol> [--path <dir>] [--kind <kind>]`.
  - Missing symbol argument fails fast with usage error.
  - Formatter output is not altered by CLI wrapper.
  - `--kind` accepts only: `function`, `method`, `class`, `interface`, `type`, `constant` (case-insensitive input).
  - Invalid kind message is exact:
    - `invalid kind "<value>" — allowed: function, method, class, interface, type, constant`
  - Ambiguous symbol message is exact:
    - `ambiguous symbol "<name>" — <N> definitions found. Use --path to narrow scope.`
  - Missing LSP binary path returns non-zero with error text containing the missing binary name and an install hint.

## Implementation Notes
- Register command in root command initialization without changing existing command behavior.
- Propagate lifecycle and fallback errors without losing required substrings.
- Missing-binary errors must preserve actionable guidance contract (include binary name and `install` substring).
- Ambiguous-symbol and invalid-kind errors must preserve exact contract strings from spec.
- Backward compatibility: ensure `index/search/status/clear/extract` remain untouched in behavior.
