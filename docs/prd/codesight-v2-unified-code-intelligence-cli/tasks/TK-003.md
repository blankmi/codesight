## Data Sources & Invariants
- Source of truth for language-to-server mapping: `pkg/lsp/registry.go`.
- Source of truth for lifecycle defaults: internal constants, `CODESIGHT_STATE_DIR`, and command inputs (no hidden global state beyond PID/state files).
- Invariants:
  - Default idle timeout is 10 minutes.
  - Lifecycle persists PID/state under `${CODESIGHT_STATE_DIR:-~/.codesight}`.
  - State keys are scoped per `(workspace root, language)`.
  - Reuse only valid live processes; stale PID state must be detected and repaired.
  - Process launch must avoid shell interpolation.
  - Runtime model is in-process child execution (same host/container as `cs`), not remote/TCP.

## Implementation Notes
- This task must define the factory/registry wiring contract that later `refs` and `callers` tasks consume.
- Registry entries must include executable name and args in a structured form.
- Lifecycle operations must be concurrency-safe for overlapping command invocations.
- Expose a deterministic state-dir resolver that honors `CODESIGHT_STATE_DIR` and defaults to `~/.codesight`.
- Backward compatibility: no side effects to index/search code paths.
- Add compile-only deferred tests when command wiring is not yet present:
  - `t.Skip("blocked by TK-006: refs command integration not wired")`
