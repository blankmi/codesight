## Data Sources & Invariants
- Source of truth for symbol lookup and references: LSP requests via TK-004 client.
- Source of truth for fallback behavior: FR-05 contract.
- Invariants:
  - Symbol resolution is name-only via `workspace/symbol`.
  - If multiple definitions match, return deterministic candidate list sorted by file path and fail with:
    - `ambiguous symbol "<name>" — <N> definitions found. Use --path to narrow scope.`
  - LSP path returns binding-accurate references for resolved symbol.
  - Fallback path includes note substring `(grep-based - install <lsp> for precise results)`.
  - Kind filter enum is fixed to: `function`, `method`, `class`, `interface`, `type`, `constant`.
  - Invalid `--kind` values fail with:
    - `invalid kind "<value>" — allowed: function, method, class, interface, type, constant`
  - Output line contract: `<relative-path>:<line>  ->  <code-snippet>`.
  - Summary line contract: `<N> references found`.
  - Output ordering is deterministic.

## Implementation Notes
- `--kind` validation and normalization are mandatory: case-insensitive input, lowercase normalized output.
- Keep grep fallback implementation isolated so LSP and fallback paths share one formatter.
- Exact output pattern and fallback note substrings are mandatory test contracts.
- Backward compatibility: this task must not yet alter CLI command registration.
