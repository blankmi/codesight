## Data Sources & Invariants
- Source of truth for hierarchy data: LSP `callHierarchy` APIs through TK-004 client.
- Source of truth for depth: input parameter validated by engine/CLI wrapper.
- Invariants:
  - Symbol resolution is name-only via `workspace/symbol`.
  - If multiple definitions match, return deterministic candidate list sorted by file path and fail with:
    - `ambiguous symbol "<name>" — <N> definitions found. Use --path to narrow scope.`
  - Default depth is `1`; invalid/non-positive depth fails with actionable error.
  - No grep fallback is allowed for callers.
  - Missing LSP binary error format is exact:
    - `cs callers: LSP required but <binary> not found. Install: <install command>`
  - Output indentation is exactly two spaces per depth level.
  - Child marker pattern is `<-`.
  - Final summary line is `<N> callers (depth <D>)`.
  - Traversal avoids infinite loops on recursive/cyclic call graphs.
