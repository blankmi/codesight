  - `index.warm_lsp` = `false`
- **Precedence invariant**: defaults < user TOML < project TOML < env vars. Each layer only overrides fields it explicitly sets; unset fields preserve the previous layer's value.
- **Env var mapping** (must match FR-02 table exactly):
  - `CODESIGHT_DB_TYPE` -> `db.type`
  - `CODESIGHT_DB_ADDRESS` -> `db.address`
  - `CODESIGHT_DB_TOKEN` -> `db.token`
  - `CODESIGHT_OLLAMA_HOST` -> `embedding.ollama_host`
  - `CODESIGHT_EMBEDDING_MODEL` -> `embedding.model`
  - `CODESIGHT_OLLAMA_MAX_INPUT_CHARS` -> `embedding.max_input_chars`
  - `CODESIGHT_STATE_DIR` -> `state_dir`
  - `CODESIGHT_GRADLE_JAVA_HOME` -> `lsp.java.gradle_java_home`
- **Missing file handling**: A missing config file at either path is silently ignored (not an error). A file that exists but contains invalid TOML returns an error.
- **Unknown keys**: Log a warning to stderr for unknown top-level sections or unknown keys within known sections. Do not abort.

## Implementation Notes

- Use `github.com/BurntSushi/toml` for TOML parsing. Add it via `go get`.
- The `Config` struct should have exported fields with `toml` struct tags for direct unmarshalling.
- Provide a `LoadConfig(projectPath string) (*Config, error)` function as the primary API.
- Provide a `Defaults() *Config` function that returns the built-in defaults.
- Track provenance (which layer set each value) for use by `cs config` in TK-005. Use a parallel `map[string]string` where key is the dotted config path and value is the source label (e.g., `"default"`, `"~/.codesight/config.toml"`, `".codesight/config.toml"`, `"CODESIGHT_EMBEDDING_MODEL"`).
- The `max_input_chars` env var contains an integer string; parse it with `strconv.Atoi` and treat non-parseable values as a warning (keep previous layer's value).
- `timeout` fields are duration strings (e.g., `"60s"`, `"90s"`). Parse with `time.ParseDuration`.
- The TOML struct should use nested structs: `Config.DB`, `Config.Embedding`, `Config.LSP.Java`, `Config.LSP.Go`, `Config.Index`.

## Unit Tests

### File: `pkg/config/config_test.go` (new)

**Coverage expectations:**
- `TestDefaults` — verify `Defaults()` returns expected default values for all fields.
- `TestLoadConfig_NoFiles` — with no config files, `LoadConfig` returns defaults.
- `TestLoadConfig_UserOnly` — user config overrides defaults; project fields stay default.
- `TestLoadConfig_ProjectOverridesUser` — project config overrides user config.
- `TestLoadConfig_EnvOverridesAll` — set env vars, verify they take precedence over both files.
- `TestLoadConfig_MalformedTOML` — returns an error on invalid TOML.
- `TestLoadConfig_MissingFileIgnored` — non-existent file paths are silently skipped.
- `TestLoadConfig_Provenance` — verify provenance map tracks correct source for each key.
- `TestLoadConfig_PartialOverride` — a config file that sets only `embedding.model` does not reset other fields.
- `TestLoadConfig_MaxInputCharsEnvParsing` — valid integer string parsed correctly; invalid string logged as warning, previous value preserved.

Use `t.TempDir()` for creating temporary config files. Use `t.Setenv()` for environment variable overrides.
