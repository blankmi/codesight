# TK-002: Wire config loading into CLI and replace env var call sites

- **Track**: Backend
- **Type**: logic
- **Modules**: `cmd/cs`, `pkg/config`

## Goal

Replace all `envOrDefault` and direct `os.Getenv("CODESIGHT_*")` calls in the CLI with config struct lookups, loading configuration once at startup.

### Allowed
- `cmd/cs/main.go`
- `cmd/cs/main_test.go`
- `pkg/config/config.go` (minor additions if needed for accessor methods)
- `pkg/config/config_test.go` (if new methods are added)

### Out-of-Scope

- FR-02 (Config Layering and Precedence)

- FR-07 (Backward Compatibility)

## Data Sources & Invariants

- **Source of truth**: The `Config` struct from `pkg/config.LoadConfig()` is the single source of truth after loading.
- **Loading point**: Config must be loaded once in `rootCmd.PersistentPreRunE` (or equivalent early hook) so all subcommands have access.
- **Project path resolution**: The project path for config loading comes from the command's path argument (or `.` if omitted). For commands that take a `[path]` argument (`index`, `search`, `status`, `clear`), use that path. For commands that don't take a path (`refs`, `callers`, `implements`), use the `--path` flag or working directory.
- **Backward compatibility invariant**: Every existing test in `cmd/cs/main_test.go` must continue to pass without modification. If any test sets `CODESIGHT_*` env vars, the config system must respect them identically.
- **Call sites to replace** (all in `cmd/cs/main.go`):
  - `envOrDefault("CODESIGHT_DB_TYPE", "milvus")` ~line 192
  - `envOrDefault("CODESIGHT_DB_ADDRESS", "localhost:19530")` ~line 195
  - `os.Getenv("CODESIGHT_DB_TOKEN")` ~line 196
  - `envOrDefault("CODESIGHT_OLLAMA_HOST", ...)` ~line 204
  - `envOrDefault("CODESIGHT_EMBEDDING_MODEL", ...)` ~line 205
  - `parseOllamaMaxInputCharsOverride()` ~line 272
  - `os.Getenv("CODESIGHT_GRADLE_JAVA_HOME")` ~line 894
- **`envOrDefault` removal**: After all call sites are migrated, remove the `envOrDefault` function and `parseOllamaMaxInputCharsOverride` function. Ensure no remaining callers exist.

## Implementation Notes

- Store the loaded `*config.Config` in a package-level variable or pass it through Cobra's context. Package-level variable is simpler given the current codebase pattern.
- The `envOrDefault` function can be removed once all call sites are replaced. The config system handles env var precedence internally.
- `parseOllamaMaxInputCharsOverride()` logic moves into `pkg/config`'s env var merge layer (already handled by TK-001).
- Do not change the behavior of any command ‚Äî only the mechanism by which config values are obtained.
- The `newVectorStore` and `newEmbedder` factory functions should accept config values as parameters rather than reading env vars directly.
