# TK-004: Add `cs init` command with project-type detection

- **Track**: Backend
- **Type**: logic
- **Modules**: `cmd/cs`

## Goal

Add a `cs init` command that scaffolds `.codesight/config.toml` with sensible defaults based on detected project type, plus `.codesight/.gitignore`.

## Files

### Allowed
- `cmd/cs/init_command.go` (new)
- `cmd/cs/init_command_test.go` (new)
- `cmd/cs/main.go` (add `init` command registration only — minimal edit)
- `pkg/config/config.go` (add template/default generation helpers if needed)
- `pkg/config/config_test.go` (if new helpers are added)

### Out-of-Scope
- `pkg/lsp/*` — no LSP changes
- `cmd/cs/config_command.go` — TK-005
- `pkg/config/config.go` core loading logic — stable from TK-001

## Specs

- FR-05 (`cs init` Command)

## Data Sources & Invariants

- **Project type detection rules** (check file existence at target path):
  | File(s) Present | Detected Type | Config Section |
  |-----------------|---------------|----------------|
  | `build.gradle.kts`, `build.gradle`, or `pom.xml` | Java | `[lsp.java]` with `gradle_java_home = ""` placeholder |
  | `go.mod` | Go | `[lsp.go]` with `build_flags = []` |
  | `Cargo.toml` | Rust | `[lsp.rust]` (empty section, placeholder) |
  | `package.json` | TypeScript/JS | `[lsp.typescript]` (empty section, placeholder) |
  | None detected | Generic | Only `[embedding]` section with model default |
- **Multiple types**: If multiple project type files are detected, include all relevant sections.
- **No-clobber invariant**: If `.codesight/config.toml` already exists, print `.codesight/config.toml already exists, skipping` to stdout and exit with code 0 (not an error).
- **Output contract**:
  - On creation: `Created .codesight/config.toml\nCreated .codesight/.gitignore\n`
  - If gitignore already exists: `Created .codesight/config.toml\n` (no gitignore line)

## Implementation Notes

- Place the command handler in `cmd/cs/init_command.go` per the file placement rules in AGENTS.md.
- Register the command in `cmd/cs/main.go` with `rootCmd.AddCommand(initCmd)`.
- The command takes one optional argument: the target path (defaults to `.`).
- Use `cobra.MaximumNArgs(1)` for argument validation.
- Generate TOML output as a formatted string (template), not by marshalling a struct. This gives control over comments and formatting in the generated file.
- Include helpful comments in the generated `config.toml` explaining each section.
- Reuse or extract the `.codesight/.gitignore` creation logic from TK-003 into a shared helper (e.g., `ensureCodesightGitignore(dir string) error`).
