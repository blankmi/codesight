# TK-005: Add `cs config` command for effective config display

- **Track**: Backend
- **Type**: logic
- **Modules**: `cmd/cs`, `pkg/config`

## Goal

Add a `cs config` command that displays the fully resolved configuration for a project path, showing each key's value and which layer set it.

## Files

### Allowed
- `cmd/cs/config_command.go` (new)
- `cmd/cs/config_command_test.go` (new)
- `cmd/cs/main.go` (add `config` command registration only — minimal edit)
- `pkg/config/config.go` (add provenance display helper if needed)
- `pkg/config/config_test.go` (if new helpers are added)

### Out-of-Scope
- `pkg/lsp/*` — no LSP changes
- `cmd/cs/init_command.go` — TK-004 (already done)

## Specs

- FR-06 (`cs config` Command)

## Data Sources & Invariants

- **Source of truth**: `LoadConfig()` from `pkg/config` provides both the resolved values and the provenance map (built in TK-001).
- **Output format invariant** (one line per key, sorted alphabetically by dotted key):
  ```
  <key> = <value> (<source>)
  ```
  Where `<source>` is one of:
  - `default`
  - `~/.codesight/config.toml`
  - `.codesight/config.toml`
  - The literal env var name (e.g., `CODESIGHT_EMBEDDING_MODEL`)
- **Keys to display**: All config keys that have a defined default or are set by any layer. Do not display internal/computed keys.
- **Empty string values**: Display as empty (e.g., `db.token =  (default)`). Do not omit them.
- **List values**: Display as comma-separated (e.g., `lsp.go.build_flags = -tags=integration`).

## Implementation Notes

- Place the command handler in `cmd/cs/config_command.go` per file placement rules.
- Register with `rootCmd.AddCommand(configCmd)`.
- The command takes one optional argument: the project path (defaults to `.`).
- Use `cobra.MaximumNArgs(1)` for argument validation.
- Iterate over the provenance map, sort keys, format output.
- The provenance map must be populated by `LoadConfig()` (TK-001 requirement).

## Unit Tests

### File: `cmd/cs/config_command_test.go` (new)

- `TestConfig_DefaultsOnly` — no config files or env vars, verify output shows all keys with `(default)` source.
- `TestConfig_WithProjectConfig` — create a `.codesight/config.toml` setting `embedding.model`, verify output shows that key with `(.codesight/config.toml)` source while others show `(default)`.
- `TestConfig_WithEnvOverride` — set `CODESIGHT_EMBEDDING_MODEL` env var, verify output shows that key with `(CODESIGHT_EMBEDDING_MODEL)` source.
- `TestConfig_OutputSorted` — verify output lines are in alphabetical order by key.
- `TestConfig_OutputFormat` — verify exact `<key> = <value> (<source>)` format.
