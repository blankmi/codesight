# TK-001: Extend lifecycle state model for daemon metadata

- **Track**: Backend
- **Type**: logic
- **Modules**: `pkg/lsp`

## Goal

Add daemon-oriented lifecycle metadata (socket path + compatibility-safe state schema) needed to support cross-invocation process reuse.

## Data Sources & Invariants

- **Source of truth for state root**: `pkg/lsp.ResolveStateDir()`.
- **Source of truth for keying**: `StateKey(canonicalWorkspaceRoot, normalizedLanguage)`.
- **State path invariant**: `<state_root>/lsp/<state_key>.json`.
- **Socket path invariant**: `<state_root>/lsp/<state_key>.sock` stored in state JSON.
- **Backward compatibility invariant**: state files without `socket_path` must still deserialize and be recoverable.
- **Permission invariant**: state files remain user-only (`0600`).

## Implementation Notes

- Keep explicit error wrapping (`fmt.Errorf("...: %w", err)`).
- Preserve existing lifecycle behavior for PID reuse/stale-PID cleanup; only extend schema and helpers needed by daemon mode.
- Corrupt/invalid state JSON must be treated as stale and removable.
- Add compatibility logic for older state payloads (missing fields) instead of hard-failing.
