## Data Sources & Invariants

- **Source of truth for warmup toggle**: effective config key `index.warm_lsp`.
- **Source of truth for workspace/language resolution**: existing refs path + language detection pipeline.
- **Platform invariant**:
  - Linux/macOS run daemon warmup path.
  - Windows keeps legacy startup path; warmup command must not introduce daemon-only runtime failures.
- **Output contract invariants**:
  - Warmup success line: `LSP warmup ready (<language>): <workspace_root>`
  - Unsupported-language line: `No supported LSP language detected for warmup`
  - Slow cold-start hint line (exact): `Tip: run 'cs warmup .' to pre-start the language server`
- **Index behavior invariant**: background warmup failure must not fail index command; warning includes `lsp warmup failed`.

## Implementation Notes

- Keep new command logic in dedicated `cmd/cs/warmup_command.go` and avoid monolithic expansion in `main.go`.
- Warmup command should share daemon connector factory wiring with refs/callers/implements runtime.
- Cold-start hint should trigger only for Java cold starts exceeding 5 seconds; warm calls, Windows legacy mode, and non-Java flows must not emit it.
- Add backward-compatibility integration tests for command wiring surfaces touched by warmup additions.
