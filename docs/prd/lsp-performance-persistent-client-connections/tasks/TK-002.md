# TK-002: Implement daemon server and JSON-RPC proxy runtime

- **Track**: Backend
- **Type**: logic

## Goal

Introduce a local daemon process that owns child language-server stdio and proxies JSON-RPC over a Unix socket for warm reuse.

## Data Sources & Invariants

- **Source of truth for launch command**: `pkg/lsp.Registry.Lookup(language)` server spec.
- **Source of truth for socket/state paths**: lifecycle state helpers from TK-001.
- **Platform invariant**:
  - Linux/macOS run Unix-socket daemon mode.
  - Windows path must compile but return deterministic daemon-disabled behavior for caller fallback.
- **Single-client invariant**: at most one active socket client per daemon instance.
- **Error contract invariant**: concurrent connect rejection must include substring `lsp daemon busy`.
- **Shutdown ordering invariant**: `shutdown` request -> `exit` notification -> forced kill fallback.

## Implementation Notes

- Preserve JSON-RPC framing and request/response IDs end-to-end.
- Remove stale socket file before listen when stale process is confirmed dead.
- Listener/setup must be safe on repeated restarts for same key.
- Guard Unix-socket implementation with platform-safe files so Windows builds do not import unsupported syscalls.
- Keep helper APIs focused; CLI-facing integration belongs to TK-004.
- If any integration test path cannot execute pre-wiring, add compiling `t.Skip("blocked by TK-004: command wiring not complete")` cases.
