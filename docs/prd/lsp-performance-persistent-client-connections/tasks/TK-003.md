## Data Sources & Invariants

- **Source of truth for server binary/args**: registry lookup output.
- **Source of truth for workspace keying**: canonical workspace root + normalized language.
- **Platform invariant**:
  - Linux/macOS connector uses daemon connect/launch path.
  - Windows connector returns explicit legacy fallback routing for command layer.
- **Retry invariant**: one stale-cleanup retry per connect attempt.
- **Client readiness invariant**: returned connector client must be initialized for immediate engine calls.

## Implementation Notes

- Add explicit factory/registry wiring API, e.g. `NewDaemonConnector(registry, opts...)`.
- Connector must not mutate registry defaults globally.
- Preserve missing-binary guidance format used by existing callers/implements errors.
- Return typed/sentinel fallback signal for unsupported daemon platforms so command wiring can switch to legacy startup without string-matching.
- If command-level tests are staged later, use compiling deferred tests with `t.Skip("blocked by TK-004: command runtime not switched")`.

## Unit Tests

- **Must update/create**:
  - `pkg/lsp/daemon_client_test.go` (new)

- Connect to existing daemon without relaunch.
- Launch daemon when no state/socket exists.
- Stale state cleanup then reconnect succeeds.
- Retry budget exhaustion returns wrapped error.
- Unsupported-platform connect path yields deterministic fallback signal.
