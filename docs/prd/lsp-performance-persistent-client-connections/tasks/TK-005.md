# TK-005: Add configurable daemon idle timeout

- **Track**: Backend
- **Type**: logic
- **Modules**: `pkg/config`, `pkg/lsp`, `cmd/cs`

## Data Sources & Invariants

- **Source of truth**:
  - Config key: `lsp.daemon.idle_timeout`
  - Env override: `CODESIGHT_LSP_DAEMON_IDLE_TIMEOUT`
  - Default: `10m`
- **Validation invariant**: value must parse as positive duration (`time.ParseDuration`, `> 0`).
- **Provenance invariant**: `cs config` output must include source label for `lsp.daemon.idle_timeout`.

## Implementation Notes

- Keep backward compatibility: unset config reproduces current `10m` behavior.
- Invalid timeout must fail config load with explicit message containing `invalid lsp.daemon.idle_timeout`.
- Wire parsed timeout into daemon startup options; avoid hidden global state.
