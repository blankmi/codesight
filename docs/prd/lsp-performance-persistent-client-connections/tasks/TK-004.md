# TK-004: Wire refs/callers/implements to daemon connector

- **Track**: Backend
- **Type**: logic
- **Modules**: `cmd/cs`, `pkg/lsp`

## Goal

Switch command runtime for refs/callers/implements from per-invocation process startup to daemon connector transport, preserving current CLI contracts.

## Files

### Allowed files

- `cmd/cs/main.go`
- `cmd/cs/lsp_runtime.go` (new)
- `cmd/cs/refs_command_test.go`
- `cmd/cs/callers_command_test.go`
- `cmd/cs/implements_command_test.go`
- `pkg/lsp/lifecycle_test.go`

### Out-of-scope files

- `pkg/config/*` (config changes are TK-005)
- `cmd/cs/warmup_command.go` (TK-007)
- `pkg/vectorstore/*`

## Data Sources & Invariants

- **Source of truth for language-server mapping**: `pkg/lsp.Registry`.
- **Source of truth for workspace resolution**: existing `resolveRefsWorkspaceRoot` path logic.
- **Platform routing invariant**:
  - Linux/macOS must use daemon connector path.
  - Windows must use legacy per-invocation startup path.
- **Contract invariants**:
  - `refs` fallback note format remains exactly `(grep-based - install <binary> for precise results)`.
  - `callers` missing-LSP error remains exactly `cs callers: LSP required but <binary> not found. Install: <hint>`.
  - `implements` missing-LSP error remains exactly `cs implements: LSP required but <binary> not found. Install: <hint>`.
  - Ambiguity header remains `ambiguous symbol "<symbol>" — <n> definitions found. Use --path to narrow scope.`

## Implementation Notes

- Keep command parsing and output formatting behavior unchanged; only transport acquisition path changes.
- Prefer new dedicated runtime file (`cmd/cs/lsp_runtime.go`) over expanding monolithic logic in `main.go`.
- Explicitly wire daemon connector creation through registry factory once per command invocation.
- For Windows, route through existing `startRefsLSPClient`-style legacy startup so behavior contracts remain unchanged.
- Remove/resolve deferred `t.Skip` cases introduced for wiring dependencies.
- Preserve explicit error wrapping style.
